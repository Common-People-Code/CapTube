## What This Project Is

CapTube is a fork of [Cap](https://cap.so) — an open-source, privacy-first screen recorder (a Loom alternative). This fork adds the ability to upload finished recordings straight to a user's own **YouTube** channel as **unlisted** videos, with the shareable link copied to the clipboard and a desktop notification.

The defining constraint: the YouTube feature is **fully self-contained** — it does not depend on cap.so's web service, account, or pricing, and the distributed binary ships **no credentials**. Each user brings their own Google OAuth client (BYOK).

**Current Phase:** Phase 3 — Notification "Copy link" action button (pending)
**Current Focus:** Phases 1 & 2 complete (PR #1). Next: a real action button on the upload notification, plus a live dogfood pass.

## Conventions & Rules

Follow `AGENTS.md` exactly. Highlights that bit us / matter most for this feature:

- **Never hand-edit generated files.** `apps/desktop/src/utils/tauri.ts` is generated by tauri-specta. Regenerate with `CAP_EXPORT_BINDINGS=1 cargo test -p cap-desktop binding_export`. It is **platform-specific** — regenerating on Linux flips `SystemDiagnostics`/`MacOSVersionInfo` to their non-macOS form; restore those two lines to the macOS form so the diff is only your change.
- **`cargo fmt --all` reformats the whole workspace.** It will touch files unrelated to your change (`fake_window.rs`, `windows.rs`, `latency.rs`, `screen_capture/mod.rs`, `timestamp/macos.rs`). Revert those before committing to keep diffs clean.
- **specta forbids `i64`/bigint in exported types** — annotate timestamp fields `#[specta(type = f64)]`.
- **TS:** tab indent, double quotes (Biome). **Rust:** clippy-clean shapes from AGENTS.md the first time.
- **Comments:** default to none; only for non-obvious context (see AGENTS.md).

### Non-negotiable quality standards
- All fallible operations have error handling; surface user-facing errors via `commands.globalMessageDialog` / toast.
- No hardcoded secrets — BYOK is the whole point. The shipped binary must contain no client ID/secret.
- No TODO comments — create a task or address it now.

## Memory & Context Notes

Non-obvious things about this codebase:
- Two recording modes differ crucially: **instant** recordings have a finished `.mp4` at stop; **studio** recordings only produce one after an editor/headless export. Auto-upload for studio must render first — which is why Phase 2 routes through the automation host (it already renders headlessly).
- The desktop `tauri-plugin-store` keeps tokens in **plaintext** (parity with the existing `auth.rs`). Keychain hardening is a deliberate later pass.
- Building `cap-desktop` needs GTK/ffmpeg/alsa system libs and the three gitignored sidecar binaries under `apps/desktop/src-tauri/binaries/` (`cap-muxer`, `cap-exporter`, `cap-cli`); a fresh sandbox lacks them.
